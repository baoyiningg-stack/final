一辆车是怎么跑起来的

AI 产品不是一个东西，而是层层装配起来的系统。点「下一步」，30 秒看完整条总装线——后面五章，每章拆一层。

STEP 1 · 模型层🔩 先造发动机大模型：会读、会写、会推理的"大脑"。但它只是发动机，不是车。例GPT-5.5、DeepSeek-V4、Qwen3.7-Max、Claude Opus、GLM-5.1——这些都是发动机的名字
STEP 2 · 产品层🚗 装进整车产品给发动机配上车身：界面、账号、工具。例ChatGPT（装的是 GPT-5.5）、豆包（装的是 Seed 2.0）、Kimi、Claude.ai——这些是车的名字
STEP 3 · 燃料⛽ 加上油Token 是油：读进来的、说出去的每个"字块"都烧油计费。例会员 ¥98/月 = 包油费的包月租车；API ¥1~36/百万token = 自己按升加油
STEP 4 · Agent层🧑‍✈️ 请上司机Agent 让车自己跑：规划路线、开车送货、出错自己调头。例Claude Code、ChatGPT Agent 模式、扣子、OpenClaw（龙虾）。没请司机时，你自己就是司机
STEP 5 · 开发者层🏭 组建车队企业通过 API 把发动机买回去，装进自己的系统组车队。例各种 App 里的"AI 助手"，背后多是接了 DeepSeek / Qwen / GPT 的 API

第 5 步：企业不满足于租车，通过 API 把发动机买回去组车队——你用的很多 App 里的"AI 助手"，背后就是这么接进去的。

❓ 我在产品里买会员，算不算"买油"？
算。会员 = 包油费的包月租车：你每次对话背后照样在烧 token，只是油钱由平台垫付、打包进了月费。所以会员都有"每小时限多少条""高峰限速"——那就是变相的油量上限。免费版则是平台补贴油费请你试驾。只有走 API，才是自己到加油站按升付油钱。
❓ 没请司机（Agent），车怎么也能开？
因为你自己就是司机。普通聊天 = 手动挡：你说一步、它动一步，查资料、做判断、把它给的代码存成文件，这些"驾驶动作"都是你在做。Agent = 把方向盘交给 AI 代驾，你只报目的地。中间还有"辅助驾驶"档（产品自带工具）——详见第四章的光谱仪。

CHAPTER 01 / 模型层

发动机：大模型本体

🔩 比喻：模型 = 发动机。它决定动力上限，但它自己跑不上路。

大模型是用海量数据训练出来的"语言大脑"：输入内容 → 计算 → 输出内容。本章三个装置，拆它的三个关键参数：吃什么油（token）、油箱多大（上下文）、有没有高性能模式（思考模式）。

装置 01Token 切词台TOKEN / 计量单位

模型不按"字"读文章，而是切成一块块 token（约 1 token ≈ 1.5~2 个汉字）。输入和输出都按 token 计费——这是 AI 世界的水表电表。输入任意文字试试：

帮我分析一下MacBookAir在京东618期间的价格竞争力，并给出三条促销建议。
切出 token 数23
这段话作为输入¥0.00002
若回复同样长度¥0.00005
这样的问答 1 万次¥0.690

※ 切分为演示用近似（每家分词器不同）。输出单价通常是输入的 3~5 倍——生成比阅读更耗算力。价格为 2026 年 6 月核对的官网价。

装置 02上下文水箱CONTEXT WINDOW / 工作记忆

上下文窗口是模型一次能"看见"的内容上限，像一个水箱。聊得越久水位越高，装满之后最早的对话被挤出去遗忘。拖动滑块模拟：

3%

一份 30 万字的报告 ≈ 18 万 token：32K 的模型装不下，1M 的可以整本吞下——这就是"长上下文"的意义。

装置 03思考模式开关THINKING / REASONING

很多模型有两种回答方式：直接答，或先在"草稿纸"上推理一遍再答（思考模式 / 推理模型）。草稿你看不见，但每个字都算 token、都收钱。拨动开关对比：

思考模式：关

QUESTION

38 个仓库要分 1200 台 MacBook，按近 30 天销量加权，但 5 个仓库的库容只剩 10 台，怎么分？

ANSWER

🧠 隐藏思考草稿（约 2,400 token，计费但默认不展示）：先算总销量权重…→ 5 个受限仓先按上限锁定 → 剩余 1150 台在 33 仓重新归一化 → 复检是否新超容 → 迭代两轮收敛…
先按销量比例直接分配即可，受限仓库分到多少就发多少。（快，但漏了"超容部分要回流重分"这一步）
耗时
~2s
token 费
×1

经验法则：简单问答关思考（快且便宜），数学 / 多约束规划 / 复杂代码开思考（贵几倍但不容易错）。

⚠️ 本章关键边界：模型本身只是"输入内容 → 输出内容"。能不能操作文件、联网、生成网页，取决于上面几层给没给它工具——这正是下一章的内容。

CHAPTER 02 / 能力层

性能配置：哪些是自带的，哪些是外挂的

⚙️ 比喻：发动机自带的性能 vs 后装的设备。最容易混淆的一层。

"AI 能生成图片 / 能联网 / 能读 PDF"——这些表现有的来自模型本身，有的来自产品外挂的工具。分不清这一点，就会误以为"DeepSeek 不会做网页"。

装置 04能力溯源台NATIVE vs TOOL-AUGMENTED

点一个你见过的"AI 能力"，看它到底来自谁、背后大概发生了什么：

🧠 模型自带 🔧 工具外挂 🌓 两种都有
背后大概发生了什么
用户输入 → 模型直接生成文字 → 输出。纯发动机性能，任何入口都有。

所以「DeepSeek 网页版只能聊天」不代表模型不会写网页——是这个入口没给它配"创建文件、运行代码、预览网页"的工具链。换个带工具的入口（比如把它的 API 接进编程工具），同一个模型立刻能交付文件。

CHAPTER 03 / 产品层

整车：你实际打开的那个入口

🚗 比喻：产品 = 整车。同一颗发动机，配置单不同，体验天差地别。

产品 = 把模型、界面、工具、账号、额度、安全策略组装在一起。拆开任何一个 AI 产品，配置单都是这六项：

模型 底层能力（发动机） 界面 聊天框、文件区（驾驶舱） 工具 搜索、代码、图片（车载设备） 账号 历史、记忆、权限（行驶证） 额度 会员、频率上限（限速器） 安全策略 能做 / 不能做（交规）
装置 05入口决策器WHICH PRODUCT?

初学者最常问的就是"我该用哪个"。点你的目标：

建议：用成品聊天产品

ChatGPT、Claude、DeepSeek、Kimi、豆包、通义都行。你买的是界面、额度、文件上传、搜索这些"整车配置"。免费版先用起来，不够再开会员。
对应：产品层 · 会员订阅

产品大致三类：综合型（ChatGPT / Claude / 豆包 / 通义——聊天+文件+图片+搜索按套餐开放）、纯净型（DeepSeek App——克制的聊天框）、工作台型（Claude Code / Codex / TRAE——重点不是聊天，而是在代码环境里干活）。

CHAPTER 04 / AGENT 层

司机：从"回答问题"到"完成任务"

🧑‍✈️ 比喻：Agent = 会自己开车送货的司机。判断标准：是"告诉你怎么做"，还是"自己做完交给你"。

装置 08Tool 与 Skill 辨析台TOOL / SKILL / MCP

上面反复出现的两个零件要分清：Tool 决定"能做什么动作"，Skill 决定"做得专业不专业"。工具是手，技能是手艺：

🔧 Tool 工具

= 手和设备 · 动作能力
Agent 能执行的具体动作：搜索、运行代码、读写文件、查数据库。没有"文件创建"工具，模型再聪明也只能把代码打在聊天框里。

📖 Skill 技能

= 操作规范 · 经验知识
预先写好的"岗位操作手册"，Agent 动手前先读。不增加任何新动作，只让产出从"随意发挥"变成"按行业标准交付"。

🔌 MCP 插口

= 标准接口 · 扩展工具清单
给 Agent 接入新工具的统一标准：日历、邮箱、数据库、内部系统都能变成它的新工具。
Tool 工具Skill 技能
本质一个可调用的能力或接口一套方法、规则、流程和经验
像什么手、脚、眼睛、外接设备说明书、SOP、岗位培训手册
例子浏览器、文件系统、终端、数据库、图像生成器Excel 处理规范、PPT 制作流程、代码审查清单
没有它会怎样AI 只能说，不能真的做AI 能做，但流程不稳、漏步骤、格式不专业

🧭 一句话：Tool 给 Agent 一双手（能不能做），Skill 给它一身手艺（做得好不好），MCP 给它一排标准插口（手还能伸多远）。
装置 09"是不是 Agent"光谱仪SPECTRUM

"ChatGPT 是不是 Agent？"——这不是开关，是光谱。点三档对比：

纯聊天：你问它答，所有"动作"靠它打字输出、你自己执行。它能把网页代码写给你看，但存成文件、打开运行都得你来。
例子：DeepSeek 官方 App、早期 ChatGPT。"DeepSeek 只能对话"的真相——不是模型不会，是这台车没配工具。
CHAPTER 05 / 开发者层

车队：把 AI 接进自己的系统

🏭 比喻：不满足于租车？通过 API 把发动机买回去，组自己的车队。

同一家 AI 公司，为什么又卖会员、又卖 API、还卖 Coding Plan？因为面向三种人。这也是"AI 公司怎么赚钱"的答案。

装置 10三种加油方式计算器会员 / API / CODING PLAN

拖动滑块改变你的月用量，看哪种付费方式最划算（按国产旗舰典型价位估算，仅示意）：

会员订阅推荐

普通用户 · 在官方 App 里聊天
¥98 /月 固定
省心，但只能在 App 里手动用，不能接进程序或编程工具。轻度日常首选。

API 按量推荐

开发者 · 接进自己的程序
¥13 /月 按量
用多少付多少（按混合单价 ¥12/百万 token 估算）。把 AI 装进自己的工具或系统，只能走这条路。

Coding Plan推荐

程序员 · 在编程 Agent 工具里用
¥99 /月 含 ~50M token
包月送大额 token（约月费 15~30 倍的量），专供 Claude Code 等工具。Agent 烧 token 凶，重度编程必选。

三者不互斥：上班用公司的 API 做产品，自己订个会员日常问答，写代码再开一份 Coding Plan。

进阶选读：企业车队的三个改装件

这三个词初学者先混个脸熟即可，等真要"把 AI 接进业务"时再深入。

RAG进阶

= 开卷考试。回答前先从你的资料库里检索相关内容，再让模型"看着资料答题"。企业知识库、客服问答的标配。

Embedding进阶

= 给资料编"语义坐标"。把文字转成向量，让系统能搜到"意思相近"而不只是"字面相同"的内容。RAG 的检索靠它。

Fine-tuning 微调进阶

= 岗前特训。用你的样例继续训练模型，让它稳定学会某种格式或风格。注意：多数知识库需求用 RAG 就够，不必微调。
